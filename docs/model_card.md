# Model Card / 模型卡

## Model Overview / 模型概览

| Item / 项目 | Description / 说明 |
|---|---|
| Model / 模型 | Ridge regression RUL estimator / Ridge regression RUL 估计器 |
| Task / 任务 | Predict remaining useful life in operating cycles / 预测设备剩余可用运行周期 |
| Inputs / 输入 | Asset cycle, operating settings, rolling sensor features, asset metadata / 设备周期、运行设置、滚动传感器特征、资产元数据 |
| Outputs / 输出 | Predicted RUL, empirical interval, risk tier, recommended action / RUL 预测、经验区间、风险等级、维护建议 |
| Intended use / 预期用途 | Decision support for maintenance prioritization and pilot planning / 支持维护优先级排序和试点规划 |
| Not intended for / 非预期用途 | Autonomous deferral of safety-critical inspections or replacement / 自动推迟安全关键检查或更换 |

## Training Data / 训练数据

The default demo uses synthetic CMAPSS-style run-to-failure sensor data generated by `scripts/generate_sample_data.py`. For public benchmark validation, prepare NASA C-MAPSS FD001 with `scripts/prepare_cmapss_fd001.py`.

默认 demo 使用 `scripts/generate_sample_data.py` 生成的 synthetic CMAPSS-style run-to-failure 传感器数据。如需公开基准验证，可使用 `scripts/prepare_cmapss_fd001.py` 准备 NASA C-MAPSS FD001 数据。

## Validation / 验证方式

The training script evaluates on a holdout set split by asset ID. This avoids evaluating on later cycles from the same assets used during training.

训练脚本按设备 ID 划分 holdout set，避免在训练过的同一设备后续周期上评估，从而降低数据泄漏风险。

Reported metrics / 输出指标：

- MAE and RMSE for RUL prediction. / RUL 预测的 MAE 和 RMSE。
- Bias to show systematic over- or under-estimation. / Bias 用于观察系统性高估或低估。
- Percentage of predictions within 20 cycles. / 预测误差在 20 个周期内的比例。
- Precision and recall for the failure horizon threshold. / 故障窗口阈值下的 precision 和 recall。

## Ethical and Operational Considerations / 伦理与运营考虑

- False negatives may delay intervention and increase service disruption or safety risk.
- False negatives 可能延迟干预，增加服务中断或安全风险。
- False positives may overload maintenance capacity.
- False positives 可能让维护资源过载。
- Recommendations should remain auditable and challengeable by trained staff.
- 模型建议应可审计，并允许受训人员提出质疑或 override。
- Thresholds must be reviewed by asset engineers and operations leads before pilot use.
- 阈值在试点前必须由资产工程师和运营负责人复核。

## Monitoring Requirements / 监控要求

- Data freshness and schema checks. / 数据新鲜度和 schema 检查。
- Sensor feature drift. / 传感器特征漂移。
- Failure horizon recall against observed outcomes. / 基于真实结果监控故障窗口 recall。
- User override and rejection rate. / 用户 override 和拒绝率。
- Asset coverage by class, route, and operating condition. / 按资产类型、线路和运行条件监控覆盖范围。
