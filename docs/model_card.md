# Model Card

## Model Overview

| Item | Description |
|---|---|
| Model | Ridge regression RUL estimator |
| Task | Predict remaining useful life in operating cycles |
| Inputs | Asset cycle, operating settings, rolling sensor features, asset metadata |
| Outputs | Predicted RUL, empirical interval, risk tier, recommended action |
| Intended use | Decision support for maintenance prioritization and pilot planning |
| Not intended for | Autonomous deferral of safety-critical inspections or replacement |

## Training Data

The default demo uses synthetic CMAPSS-style run-to-failure sensor data generated by `scripts/generate_sample_data.py`. For public benchmark validation, prepare NASA C-MAPSS FD001 with `scripts/prepare_cmapss_fd001.py`.

## Validation

The training script evaluates on a holdout set split by asset ID. This avoids evaluating on later cycles from the same assets used during training.

Reported metrics:

- MAE and RMSE for RUL prediction.
- Bias to show systematic over- or under-estimation.
- Percentage of predictions within 20 cycles.
- Precision and recall for the failure horizon threshold.

## Ethical and Operational Considerations

- False negatives may delay intervention and increase service disruption or safety risk.
- False positives may overload maintenance capacity.
- Recommendations should remain auditable and challengeable by trained staff.
- Thresholds must be reviewed by asset engineers and operations leads before pilot use.

## Monitoring Requirements

- Data freshness and schema checks.
- Sensor feature drift.
- Failure horizon recall against observed outcomes.
- User override and rejection rate.
- Asset coverage by class, route, and operating condition.
